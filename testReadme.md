<h1><a id="user-content-specify-a-face-recognition-model" class="anchor" aria-hidden="true" href="#specify-a-face-recognition-model"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Specify a face recognition model</h1>
<p>This guide demonstrates how to specify a face recognition model for face detection, identification and similarity search using the Azure Face API.</p>
<p>The Face API uses deep learning models to perform operations on human faces in images. We continue to improve the accuracy of our models based on customer feedback and advances in deep learning research, and we deliver these improvements as model updates. Developers have the option to specify which version of face recognition model they'd like to use; they can choose the model that best fits their use case.</p>
<p>If you are a new user, we recommend you use the latest model. Read on to learn how to specify it in different Face operations. If you are an advanced user and are not sure whether you should switch to the latest model, skip to the <a href="#Evaluate-different-models">Evaluate different models</a> section to evaluate the new model and compare results using your current data set.</p>
<h2><a id="user-content-prerequisites" class="anchor" aria-hidden="true" href="#prerequisites"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Prerequisites</h2>
<p>You should be familiar with the concepts of AI face detection and identification. If you aren't, see these how-to guides first:</p>
<ul>
<li><a href="/MicrosoftDocs/azure-docs-pr/blob/db02cb39050636f33a36e6e7fc9b20e9ebb225c6/articles/cognitive-services/Face/Face-API-How-to-Topics/HowtoDetectFacesinImage.md">How to detect faces in an image</a></li>
<li><a href="/MicrosoftDocs/azure-docs-pr/blob/db02cb39050636f33a36e6e7fc9b20e9ebb225c6/articles/cognitive-services/Face/Face-API-How-to-Topics/HowtoIdentifyFacesinImage.md">How to identify faces in an image</a></li>
</ul>
<h2><a id="user-content-detect-faces-with-specified-model" class="anchor" aria-hidden="true" href="#detect-faces-with-specified-model"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Detect faces with specified model</h2>
<p>Face detection identifies the visual landmarks of human faces and finds their bounding-box locations. It also extracts the face's features and stores them for use in identification. All of this information forms the representation of one face.</p>
<p>The recognition model is used when the face features are extracted, so you can specify a model version when performing the Detect operation.</p>
<p>When using the <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d" rel="nofollow">Face - Detect</a> API, assign the model version with the <code>recognitionModel</code> parameter.</p>
<p>Available values:</p>
<ul>
<li><code>recognition_v01</code></li>
<li><code>recognition_v02</code></li>
</ul>
<p>So, a request URL for the <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d" rel="nofollow">Face - Detect</a> API will look like this:</p>
<pre><code>https://westus.api.cognitive.microsoft.com/face/v1.0/detect[?returnFaceId][&amp;returnFaceLandmarks][&amp;returnFaceAttributes][&amp;recognitionModel]
&amp;subscription-key=&lt;Subscription key&gt;
</code></pre>
<p>If you are using the client library, you can assign the value for <code>recognitionModel</code> by passing a string representing the version.
If you leave it unassigned, the default model version (<em>recognition_v01</em>) will be used.</p>
<div class="highlight highlight-source-cs"><pre><span class="pl-k">string</span> <span class="pl-smi">imageUrl</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>http://news.microsoft.com/ceo/assets/photos/06_web.jpg<span class="pl-pds">"</span></span>;
<span class="pl-k">var</span> <span class="pl-smi">faces</span> <span class="pl-k">=</span> <span class="pl-k">await</span> <span class="pl-smi">faceServiceClient</span>.<span class="pl-en">DetectAsync</span>(<span class="pl-smi">imageUrl</span>, <span class="pl-c1">true</span>, <span class="pl-c1">true</span>, <span class="pl-s"><span class="pl-pds">"</span>recognition_v02<span class="pl-pds">"</span></span>);</pre></div>
<h2><a id="user-content-identify-faces-with-specified-model" class="anchor" aria-hidden="true" href="#identify-faces-with-specified-model"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Identify faces with specified model</h2>
<p>When the Face API extracts face feature data from an image, the service builds a <strong>PersonGroup</strong> object for person identification.</p>
<p>A <strong>PersonGroup</strong> should have one unique recognition model for all of the Persons, and you can specify this using the <code>recognitionModelType</code> parameter when you create the group (<a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395244" rel="nofollow">PersonGroup - Create</a> or <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/599acdee6ac60f11b48b5a9d" rel="nofollow">LargePersonGroup - Create</a>).</p>
<p>See the following code example for the .NET client library.</p>
<div class="highlight highlight-source-cs"><pre><span class="pl-c"><span class="pl-c">//</span> Create an empty PersonGroup with "recognition_v02" model</span>
<span class="pl-k">string</span> <span class="pl-smi">personGroupId</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">"</span>myfriends<span class="pl-pds">"</span></span>;
<span class="pl-k">await</span> <span class="pl-smi">faceServiceClient</span>.<span class="pl-en">CreatePersonGroupAsync</span>(<span class="pl-smi">personGroupId</span>, <span class="pl-s"><span class="pl-pds">"</span>My Friends<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>recognition_v02<span class="pl-pds">"</span></span>);</pre></div>
<p>In this code, a <strong>PersonGroup</strong> with id <code>myfriends</code> is created, and it is set up to use the new <em>recognition_v02</em> model to extract face features.</p>
<p>Correspondingly, you need to specify which model to use when identifying faces against this <strong>PersonGroup</strong> (through the <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d" rel="nofollow">Face - Detect</a> API). The model you use should always be consistent with the <strong>PersonGroup</strong>'s configuration; otherwise, the operation will fail due to incompatible models.</p>
<p>There is no change in the <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395239" rel="nofollow">Face - Identify</a> API; you only specify the model version in detection.</p>
<h2><a id="user-content-find-similar-faces-with-specified-model" class="anchor" aria-hidden="true" href="#find-similar-faces-with-specified-model"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Find similar faces with specified model</h2>
<p>You can also specify a recognition model for similarity search. You can assign the model version with <code>recognitionModel</code> when creating the face list with <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f3039524b" rel="nofollow">FaceList - Create</a> API or <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/5a157b68d2de3616c086f2cc" rel="nofollow">LargeFaceList - Create</a>.</p>
<p>See the following code example for the .NET client library.</p>
<div class="highlight highlight-source-cs"><pre><span class="pl-k">await</span> <span class="pl-smi">faceServiceClient</span>.<span class="pl-en">CreateFaceListAsync</span>(<span class="pl-smi">faceListId</span>, <span class="pl-s"><span class="pl-pds">"</span>My face collection<span class="pl-pds">"</span></span>, <span class="pl-s"><span class="pl-pds">"</span>recognition_v02<span class="pl-pds">"</span></span>);</pre></div>
<p>This code creates a face list called <code>My face collection</code>, using the <em>recognition_v02</em> model for feature extraction. So, when you search for similar faces in this face list with a <code>faceId</code>, that <code>faceId</code>'s corresponding face should also be detected with the <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d" rel="nofollow">Face - Detect</a> API using <em>recognition_v02</em>. As in the previous section, the model needs to be consistent.</p>
<p>There is no change in the <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395237" rel="nofollow">Face - Find Similar</a> API; you only specify the model version in detection.</p>
<h2><a id="user-content-evaluate-different-models" class="anchor" aria-hidden="true" href="#evaluate-different-models"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Evaluate different models</h2>
<p>If you'd like to compare the performances of the <em>recognition_v01</em> and <em>recognition_v02</em> models on your data, you will need to:</p>
<ol>
<li>Create two <strong>PersonGroup</strong>s with <em>recognition_v01</em> and <em>recognition_v02</em> respectively.</li>
<li>Use your image data to detect faces and register them to <strong>Person</strong>s for these two <strong>PersonGroup</strong>s, and trigger the training process with <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395249" rel="nofollow">PersonGroup - Train</a> API.</li>
<li>Test with <a href="https://westus.dev.cognitive.microsoft.com/docs/services/563879b61984550e40cbbe8d/operations/563879b61984550f30395239" rel="nofollow">Face - Identify</a> on both <strong>PersonGroup</strong>s and compare the results.</li>
</ol>
<p>If you normally specify a confidence threshold (a value between zero and one that determines how confident the model must be to identify a face), note that you may need to use different thresholds for different models. A threshold for one model is not meant to be shared to another and will not necessarily produce the same results.</p>
